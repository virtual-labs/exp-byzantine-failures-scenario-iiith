üîç Understanding Node Types

  • üü¢ Honest Node (Green): Follows the consensus protocol correctly, always tells the truth.
  • üî¥ Byzantine Node (Red): Malicious node that may lie, send conflicting messages, or remain silent.
  • üü° Leader Node (Gold): Initiates consensus by proposing a value. Must be an honest node.
  • ‚ö´ Offline Node (Gray): Currently not used, but represents crashed/unreachable nodes.
  • Node Values: Each node starts with a random binary value (0 or 1) that they want the network to agree on.

üìã Byzantine Agreement Process

  1. Phase 1 - Leader Proposal:
    • Leader broadcasts its proposed value to all nodes
    • All nodes receive the proposal (shown as moving messages)
  2. Phase 2 - Node Responses:
    • Each node responds with their vote (accept/reject proposal)
    • Honest nodes follow protocol rules
    • Byzantine nodes may send conflicting or false responses
  3. Phase 3 - Final Decision:
    • Leader counts votes from all responses
    • Consensus reached if majority of honest nodes agree
    • Result announced to network
